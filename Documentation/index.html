1) Introduction - top

Most of the times in programming we have the need to output data of different types, which are just declared somewhere or they come from a process.Here comes the question of how we will represent them to the end user,in which format and how to manipulate the free space.Here UIjson.js came out.UIjson.js is a jquery plug-in which offers some predefined ways to render,edit and create data from JSON models.It makes use of most popular HTML, CSS, framework: BOOTSTRAP for interface and Javascript,Jquery for functionality.A lot of people love Boostrap because it makes them to not worry about cross browser/platform css compability issues. So I decided to use it here to keep you still free of worries. For the same reason I used Jquery.Both have great support, years of life ,great documentations , stability and many places to ask if you need any help. Stability is a preference for most of the clients. So lets continue to the point.

b) Understanding Dataholders

The main subject of UIjson are the properties which hold data.These 'dataholder' properties can have value of type string,number,boolean or an array of these types.See dataholders of 'example' in RED highlight. These are the properties which can be handled by UIjson. Blue properties will be Ignored 

var example  = {
       name      : "Julian",
       age       : "20",
       country   : "USA",
       past_jobs : ["Philips","ikea","apple","microsoft"],
       children  : [ {name:""},{name:""} ],
       job       : {}
};

Notice : Dataholders properties can have Interface from bootstrap. BLUE properties will be ignored from UIjson.

c) Defining Dataholder properties

To configure UIjson you will need to define a dataholder object for each property you want to import. Undefined properties will not be removed as they are part of your model, but you will not able to render or edit them.They will be ignored from all UIjson's functionality. 

//To define a dataholder Object you need at least to set these 2 properties, 'property' and 'assignTo'

var dataHolder = { 

   property:"name" , 
   assignTo:"input_field" 
};

//And here the full options of dataholder Object

var dataHolder = { 

   property  :"name" ,        // the property
   path      : "",            // Path may be used when you add a nested object which is part of a bigger instance, to separate this dataholder from others if they have same properties.
   assignTo  :"input_field"   // How this property will be rendered in HTML.There a list of UI types in code architecture section of documentation.
   tabName   :"Bio"  ,        // Default tab is 'General'. Only if tab with label 'Bio' is active you will be able to see the UI element for this property.
   label     :"Person's Name" // Label text goes to 'control-label' element of bootstrap.
   options   :[],             // Options required if assignTo is "select_list", "radio" or "checkbox"
   arrEach   : false,         // If true means that if the value of the object is an array of strings,each element of the array will be a seperate UI element in HTML
   array_Api : {              // It has meaning only if arrEach = true 
   
                sortable : true ,   // If true You can sort elements. This requires jquery-ui.
                add      : true ,   // If true You can add new elements by clicking a symbol.
                remove   : true     // If true You can remove an UI element from the array by clicking a symbol.
				
   }  
};

Notice : Each defined property is unique and Uniqueness is based on 'property' combined with 'path'. Default values are included in 'defaults' section of Documentation.

5) Progress - top

So how it works.
i ) You define some properties in configuration or using defineProp( .. ) method. The objects you use to define them including some other properties are the Dataholder Objects of UIjson.js.
All dataholder Objects are stored in Dataholders array of UIjson Instance.
ii ) You import some Objects using _Objects in configuration or using ImportObject( .. ) method. Lets say you import the object var AA = { };
Once imported , AA will be converted to UIjson Objectand can be used from UIjson methods and events.
Below you can see how UIjson Objects look like : 

var UIjsonObject = 						  
{						  
obj         : exampleOb, // A reference to the Imported Object.
ui_wrapper  : null,      // It is a Jquery element and it will not be null only if this Object is rendered.
dataholders : [],        // This array is filled from Obj_initDataholders( .. ) method
path        : ""         // comes from user's definition
}

All UIjson Objects exist in _Objects array of the Instance.

iii ) Filling UIjson_Object.dataholders array.

All properties of AA will be compared by name and path with the defined Dataholders.Matches will be pushed to UIjson_Object.dataholders array

iv ) Now everytime AA is going to be rendered, some objects of UIjson_Object.dataholders array will have "elements" property filled with some Jquery Selectors.

6) Methods - top

Method : destroy()

Destroys UIjson instance,removes all events and all elements

$("#MyUI").UIjson("destroy");
Arguments
none

Method : defineTab( str )

Defines a tab

$("#MyUI").UIjson("defineTab","SomeTabName");
Arguments
str:string

Method : tabOrder ( arr )

Changes order of existing tabs.Accepts an array of existing tab names

// Lets define some new tabs for this example								   
$("#MyUI").UIjson("defineTab","TabName1");	
$("#MyUI").UIjson("defineTab","TabName2");
$("#MyUI").UIjson("defineTab","TabName3");								   
								   
	 						   
$("#MyUI").UIjson( "tabOrder" , [ "General" , "TabName2" , "TabName1" , "TabName3" ] );
Arguments
arr:Array of strings

Method : removeTab( str )

Removes an existing Tab. If this tab is already assigned to a dataholder object, tab,section and tabName properties of this dataholder change to default.

// Lets create a new tab and then we will delete it
$("#MyUI").UIjson( "defineTab" , "SomeTabName" );							   
								   
$("#MyUI").UIjson( "removeTab" , "SomeTabName" );
Arguments
str:string

Method : defineProp( Obj )

Creates a dataholder Object for a property and pushs it to Dataholders array of UIjson instance. This method runs internally on configuration and can be used also to define dataholder Objects after configuration.

$("#MyUI").UIjson( "defineProp" , { property:"name" , assignTo:"input_field" , label:"A Name" } );
Arguments
Obj:Object

Method : getProp( str , path )

Returns a defined dataholder Object based on property name and path

var dataHolderObject = $("#MyUI").UIjson( "getProp" , "name" );
Arguments
str :String path :String

Method : propTab( dataholderObject , tabName )

Assigns tab to a dataholder object. If tabName is provided and is not defined in config,it will be defined, else if tabName is not provided it will return the assigned tab of this dataholder object.

// lets get the dataholder Object of the defined property 'name' and then run the method to change its tab.
var dataHolderObject = $("#MyUI").UIjson( "getProp" , "name" );

$("#MyUI").UIjson( "propTab" , dataHolderObject , "Name's Tab"  );
Arguments
dataholderObject:Object tabName:String Optional

Method : unsetProp( dataholderObject )

Deletes a dataholder object from Dataholders array of UIjson instance. If a UIjson Object includes this dataholder in its dataholders array, it will be removed from there too.

// Get dataholder Object of 'name' property and then run 'unsetProp' method to remove the dataholder Object
var dataHolderObject = $("#MyUI").UIjson( "getProp" , "name" );
$("#MyUI").UIjson( "unsetProp" , dataHolderObject );
Arguments
dataholderObject:Object

Notice : Before unset progress you need to use ObjAll_Clear() or Obj_ClearRender( UIjsonObj ) or Obj_PropClear( UIjsonObj , propName ) to clear affected UIjson objects from UI. After unseting progress you need to use ObjAll_Render() or Obj_Render( UIjsonObj ) or Obj_PropRender( UIjsonObj , propName ) to render UI again. This is a REQUIRED step to avoid memory leaks or other unexpected behavior

Method : ImportObject( Obj , path )

Converts Obj to UIjson Object and push it in _Objects array of the instance.

// Lets import object AA 								   
var AA = {name:"Alice"};
$("#MyUI").UIjson( "ImportObject" , AA , "" );
Arguments
Obj:Object path:String

Method : getUIjsonObj( Obj )

Returns UIjson object for an imported Obj

var UIjsonObject =    $("#MyUI").UIjson( "getUIjsonObj" , exampleOb );
console.log(UIjsonObject);
Arguments
Obj:Object

Method : Obj_initDataholders ( UIjsonObj )

Initialises dataholders of the given UIjson Object. This method runs internally only on ImportObject. It may be usefull to you if you unset or change the global dataholders.

// unset "Name"								   
var dataHolderObject = $("#MyUI").UIjson( "getProp" , "name" );
$("#MyUI").UIjson( "unsetProp" , dataHolderObject );

// Now redefine it
$("#MyUI").UIjson( "defineProp" , { property:"name" , assignTo:"input_field" , label:"A Name" } );

// Then run "Obj_initDataholders" to re initialize.
								   
var UIjsonObject =  $("#MyUI").UIjson( "getUIjsonObj" , exampleOb );
$("#MyUI").UIjson( "Obj_initDataholders" , UIjsonObject );
Arguments
UIjsonObj:UIjson Object

Method : Obj_Delete( Obj )

Deletes an imported Object and deletes it from _Objects array of the instance.

// Import Object AA and then get UIjsonObject of AA. finally delete it using "Obj_Delete" method
var AA = {name:"Alice"};
$("#MyUI").UIjson( "ImportObject" , AA , "" );
var UIjsonObjAA = $("#MyUI").UIjson( "getUIjsonObj" , AA );
								   
var UIjsonObject = $("#MyUI").UIjson( "Obj_Delete" , UIjsonObjAA );
Arguments
Obj:Object

Method : Obj_Elements( UIjsonObj , property )

Returns all the visible elements in UI of a UIjson Object. Argument UIjsonObj can be obtained from getUIjsonObj method.If property provided it will return an array of UI elements assigned to that property


var UIjsonObject = $("#MyUI").UIjson( "getUIjsonObj" , exampleOb );
var UI_elements  = $("#MyUI").UIjson( "Obj_Elements" , UIjsonObject , "name" );
Arguments
UIjsonObj:Object property :String Optional

Method : Obj_Update( UIjsonObj )

Updates UIjson object with values from interface. Argument can be obtained from getUIjsonObj method

// get 	UIjsonObject of exampleOb							   
var UIjsonObject = $("#MyUI").UIjson( "getUIjsonObj" , exampleOb );

// ..
// change Interface values of inputs etc with your own DOM methods for example.     
// ..
// Then Update UIjsonObject with new values

 $("#MyUI").UIjson( "Obj_Update" , UIjsonObject );
Arguments
UIjsonObj:Object

Method : ObjAll_Render()

Renders UI elements with values of ALL imported UIjson Objects

								   
$("#MyUI").UIjson( "ObjAll_Render" );
Arguments
None

Method : ObjAll_Clear()

Remove all rendered UI elements and events of ALL imported UIjson Objects

								   
$("#MyUI").UIjson( "ObjAll_Clear" );
Arguments
None

Method : Obj_ClearRender( UIjsonObj )

Remove all UI elements(if exist) from interface of an imported Object. Argument can be obtained from getUIjsonObj method

								   
var UIjsonObject = $("#MyUI").UIjson( "getUIjsonObj" , exampleOb );
$("#MyUI").UIjson( "Obj_ClearRender" , UIjsonObject );
Arguments
UIjsonObj:UIjson Object

Method : Obj_Render( UIjsonObj )

Renders UIjsonObj Object with its values to tab. Argument UIjsonObj can be obtained from getUIjsonObj method

								   
var UIjsonObject = $("#MyUI").UIjson( "getUIjsonObj" , exampleOb );
$("#MyUI").UIjson( "Obj_Render" , UIjsonObject );
Arguments
UIjsonObj:UIjson Object tab:String

Method : Obj_Redraw( UIjsonObj )

Re-Renders an UIjson Object. It is a shortcut method of Obj_ClearRender followed by Obj_Render

								   
var UIjsonObject = $("#MyUI").UIjson( "getUIjsonObj" , exampleOb );
$("#MyUI").UIjson( "Obj_Redraw" , UIjsonObject );
Arguments
UIjsonObj:UIjson Object

Method : Obj_PropRender( UIjsonObj , propName )

Creates UI element for propName of UIjsonObj.This method is called from Obj_Render method.

							   
var UIjsonObject = $("#MyUI").UIjson( "getUIjsonObj" , exampleOb );

$("#MyUI").UIjson( "Obj_PropRender" , UIjsonObject , "name" );
Arguments
UIjsonObj:UIjson Object propName:String

Method : Obj_PropClear( UIjsonObj , propName )

Removes assigned UI element for propName of UIjsonObj from Interface.

							   
var UIjsonObject = $("#MyUI").UIjson( "getUIjsonObj" , exampleOb );

$("#MyUI").UIjson( "Obj_PropClear" , UIjsonObject , "name" );
Arguments
UIjsonObj:UIjson Object propName:String

Method : Obj_PropRedraw ( UIjsonObj , propName )

Re-renders property "propName" of UIjsonObj. 

							   
var UIjsonObject = $("#MyUI").UIjson( "getUIjsonObj" , exampleOb );

$("#MyUI").UIjson( "Obj_PropRedraw" , UIjsonObject , "name" );
Arguments
UIjsonObj:UIjson Object propName:String

Method : Obj_PropUpdate( UIjsonObj , propName )

Updates property value of UIjsonObj based on values of Interface

							   
var UIjsonObject = $("#MyUI").UIjson( "getUIjsonObj" , AA );

$("#MyUI").UIjson( "Obj_PropRender" , UIjsonObject , "name" );
Arguments
UIjsonObj:UIjson Object propName:String

7) Events - top

Event : on Obj_Update( e , Obj )

Triggers when a value of a property has been changed from UI elements.

$("#MyUI").on("Obj_Update.UIjson",function( e , Obj ){});
Arguments
e : Jquery event Obj : UIjson Object

Event : on Obj_Render( e , Obj )

Triggers when all defined properties of Obj are rendered.

$("#MyUI").on("Obj_Render.UIjson",function( e , Obj ){});
Arguments
e : Jquery event Obj : UIjson Object

Event : on Obj_ClearRender( e , Obj )

Triggers when all defined properties of Obj are removed from UI.

$("#MyUI").on("Obj_ClearRender.UIjson",function( e , Obj ){});
Arguments
e : Jquery event Obj : UIjson Object

Event : on Obj_PropRender( e , Obj , dthdObject , el )

Triggers everytime an element is created for a UIjson Object property

$("#MyUI").on("Obj_PropRender.UIjson",function( e , Obj , propertyObj , el ){});
Arguments
e : Jquery event Obj : UIjson Object dthdObject : dataholderObject el : Jquery element

Event : on Obj_PropClear( e , Obj , dthdObject , el )

Triggers everytime an assigned element for a property of a UIjson Object is going to be removed from Interface. It runs exactly before the removal of el. This method is usefull when you have attached your own events using the api to an element and you need to detach them before the removal. 

$("#MyUI").on("Obj_PropClear.UIjson",function( e , Obj , propertyObj , el ){});
Arguments
e : Jquery event Obj : UIjson Object dthdObject : dataholderObject el : Jquery element

Event : on ObjAll_Render( e , instance )

Triggers when all defined properties of ALL Objects have finished rendering.

Arguments
e : Jquery event instance : UIjson instance

Event : on ObjAll_Clear( e , instance )

Triggers when all defined properties of ALL Objects have finished rendering.

Arguments
e : Jquery event instance : UIjson instance

Event : on createUiElement( e , UIjsonObj , dthd , el )

Triggers everytime an UI element is created.

Arguments
e : Jquery event UIjsonObj : UIjson Object dthd : dataholder Object el : Jquery element

Event : on Obj_PropUpdate( e , UIjsonObj , propName )

Triggers everytime a property of an UIjsonObj changes

Arguments
e : Jquery event UIjsonObj : UIjson Object propName : String

8) Code Architecture - top

This is the plugin constructor

$.UIjson = function (){
  this.Settings        = "";   //Merges defaults with changed defaults from user.  
  this.Element         = null; //Holds Jquery selector of UIjson instance  
  this._Objects        = [];   //Holds all UIjson objects  
  this.ActiveTab       = "";   //Keeps name of the active tab 
  this.Tabs            = {};   //Holds tab objects.It has same properties with Sections below.  
  this.Sections        = {};   //Holds section objects.It has same properties with Tabs above.  
  this.Dataholders     = [];   //Holds all the defined properties from configuration or from defineProp method.
}

Pool of UI elements

All UI elements exist as properties in UI_Elements_Pool. Each name must be unique.
_HTML contains HTML of element as string
Val_Method sets or returns value of target element.
AttachEv attach events
detachEv removes attached events

input_field : {
_HTML      : '<div class="col-md-6 col-lg-6 col-sm-6"><div class="form-group"><label class="control-label"></label><input class="form-control" type="text" </div></div>',
Val_Method : function( _HTML , value ){var target = _HTML.find("input"); if( $.type( value )!=="undefined" ){  target.val( value ); }else{ return target.val();}},
attachEv   : function( UIjsonInstance , Obj , el ){  el.find("input").eq(0).on("change",{UIjsonInstance:UIjsonInstance,Obj:Obj},$.UIjson.ActionsA.updateObj );return el;},
detachEv   : function( UIjsonInstance , el ){ el.find("input").eq(0).off("change", $.UIjson.ActionsA.updateObj  );}	 
}

UIjson supports these elements using bootstrap's classes. The below names go as value to assignTo property

  1. input_field
  2. textarea_field
  3. select_list
  4. checkbox
  5. radio
  6. thumbnail
  7. blockquote

Notice : Using this format you can add your own elements and bind them with your Model. All UI_elements are parts of UI_object so each one must have a unique name which is used to 'assingTo' property.

$.JsonEditor.Tabs

- tab_HTML -
'<li role="presentation"><a data-toggle="tab" href=""></a></li>'
This is HTML as string of tab element
- createTab -
Function to create a tab Object. Tab objects contain 'element':Jquery selector and 'name': the name of tab . Tab Objects stored in "Tabs"of UIjson constructor.
- deleteTab -
Function to unset a tab object from Tabs of UIjson instance.
- tabOrder -
Function to change order of tabs
- attachEv -
Attachs Click event to a tab element
- detachEv -
Detach Click event from a tab element

$.JsonEditor.Sections

- section_HTML -
'<div role="tabpanel" class="tab-pane tabSection fade in row" style="min-height:300px;"></div>'
- createSection -
Function to create a section object as pair to a tab. Section object contains properties "name":the name of tab and "element":Jquery selector
- deleteSection -
Function to unset A section object from Sections of UIjson instance.

9) Styling Classes - top

All the known classes of bootstrap are there.Although some elements with critical meaning have some additional classes.

.path-label
: is the className of rendered path element of an UIjson Object
.UIjson_ObjWrapper
: is the parent of all rendered UI elements of an UIjson Object.
.UIjsonArraySortable
: is the parent of all rendered UI elements which represent an array. It is applied only if arrEach is set to true.
.UIjson_el
: is the className of all rendered UI_elements as defined in $.UIjson.UI_Elements object.
.tabSection
: is the className of each section as direct child of defined section parent in config.Sections are there even if they are empty.But only the active section is visible.
